# Security Policy

This repository is a formal methods library. Security-sensitive issues are most likely to involve:

- dependency or supply-chain configuration
- CI or release automation
- credential leakage

Please do not open a public issue first for security-sensitive reports. Use GitHub's private
security reporting flow if it is available for this repository, or contact the maintainers
through GitHub directly.

Ordinary theorem mistakes, proof gaps, or documentation errors should be reported through the
normal issue tracker.
